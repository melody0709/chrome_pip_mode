# 更新日志

## [1.0.2] - 2026-04-21

### Changed
- 修复了档位检测逻辑，使用浏览器物理窗口尺寸计算，避免系统 DPI 和页面缩放导致的错乱
- 调整 MAX_VIEWPORT_RATIO 从 0.38 增加到 0.50
- 优化档位阈值，使得多屏幕/半屏判定更合理（MEDIUM 档位改为 50%-70%）
- 修复了调整窗口大小时，不同档位的缩放比例会被错误覆盖互相污染的问题
- 修复了窗口或网页缩放时，停靠在右侧/底侧的悬浮窗无法保持原有边距距离的问题
- 升级本地存储结构至 v6 版本并提供自动迁移

本文件记录项目的所有重要变更。

## [1.0.1] - 2026-04-20

### 新增
- **多档位宽度比例**：基于屏幕物理宽度百分比实现四种视窗档位（MAX/WIDE/MEDIUM/SMALL）。
  - MAX（85%-100%）：全屏模式
  - WIDE（70%-85%）：宽屏窗口模式
  - MEDIUM（55%-70%）：中等/分屏模式
  - SMALL（≤55%）：小窗口模式
- **独立比例记忆**：每个档位维护自己的 `widthRatio`，允许不同窗口大小使用不同的浮窗尺寸。
- **存储模式 v5**：从 v4 升级到 v5 以支持多档位宽度比例，并支持从 v3/v4 自动迁移。

### 变更
- **自适应宽度计算**：浏览器窗口调整大小时，浮窗现在使用对应档位的 `widthRatio`，而非单一全局比例。
- **默认比例**：为每个档位设置默认比例（MAX: 20%，WIDE: 22%，MEDIUM: 25%，SMALL: 30%）。

## [1.0.0] - 2026-04-20

### 变更
- **几何优化**：将 `widthRatio` 从边界约束中解耦，确保播放器在窗口调整后仍能"记住"用户定义的相对大小。
- **缩放适配**：使用 `window.devicePixelRatio` 实现动态 `MIN_WIDTH` 计算。防止在高缩放级别（如 500%）时播放器物理尺寸过大。
- **垂直对齐**：修复放大时悬浮播放器向上漂移的问题。现在严格保持与底部的距离。
- **层级与裁剪（Bilibili）**：为 Bilibili 实现递归祖先 `z-index` 和 `overflow` 修改策略。悬浮播放器现在能正确覆盖所有页面元素，包括侧边栏按钮（客服等）。
- **CSS 重置**：注入 `min-width: 0 !important` 和 `min-height: 0 !important` 以覆盖站点特定的容器约束，这些约束在高缩放级别会导致宽高比失真。
- **重构**：将祖先可见性逻辑提升为共享工具函数，以便更好地集成站点特定功能。
